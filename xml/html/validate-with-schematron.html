Validate with Schematron

Upload one or two instance documents and a Schematron schema to perform validation.

  • If you upload two documents, they will each be wrapped in <c:file xmlns:c="http://www.w3.org/ns/xproc-step"> elements and these two c:file elements will themselves be wrapped in a root c:directory element, which will be passed to the Schematron validator.
  • If you upload a single document, it will be validated as is.

The service returns data in two different formats, depending on whether validation succeeds or fails.

  • If validation succeeds, the service returns a response entity which is an XML document in SVRL format, with an HTTP status code of 200 (OK)
  • If validation fails, the response is a description of the errors in JSON format, with an HTTP status code of 400 (Bad Request). The JSON data is intended to be parsed into a JavaScript object and thrown as an error.
